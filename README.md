# docker-fastapi

A production-grade **Python 3.12 + FastAPI + Uvicorn** REST API starter image,
part of the branded Docker Hub starter-image family by **Lê Đức Minh**
(`minhle202`). Pull it, run it, and you have a clean, non-root, multi-stage
FastAPI service with health checks and interactive OpenAPI docs in seconds.

## Pull

```sh
docker pull minhle202/fastapi
```

## Run

```sh
docker run --rm -p 8000:8000 minhle202/fastapi
```

The API listens on port **8000** (override with the `PORT` env var).

## Try it

```sh
curl http://localhost:8000/
curl http://localhost:8000/health
curl "http://localhost:8000/api/hello?name=Minh"
```

Interactive API docs (Swagger UI, auto-generated by FastAPI) are served at
**http://localhost:8000/docs**.

## Endpoints

| Method | Path               | Description                                                        |
| ------ | ------------------ | ------------------------------------------------------------------ |
| GET    | `/`                | Service metadata: `name`, `version`, and a list of `endpoints`.    |
| GET    | `/health`          | `{ "status": "ok", "uptime": <seconds since start> }`.             |
| GET    | `/api/hello?name=` | `{ "message": "Hello, <name>!" }`. `name` defaults to `"world"`.   |
| GET    | `/docs`            | Interactive Swagger UI (OpenAPI), auto-generated by FastAPI.       |

## What's inside

- **Python 3.12-slim** base image — small, current, secure.
- **FastAPI + Uvicorn** — modern async web framework on a production ASGI server.
- **Multi-stage build** — dependencies installed into an isolated virtualenv in a
  builder stage, then only `/opt/venv` is copied into the slim runtime image.
- **Non-root user** (`appuser`, uid 10001) for least-privilege runtime.
- **HEALTHCHECK** that polls `/health` so orchestrators know when the app is ready.
- **Interactive docs** at `/docs` (and the raw OpenAPI schema at `/openapi.json`).
- Uvicorn handles **SIGTERM** gracefully, so the container shuts down cleanly on
  `docker stop` with no extra signal-handling code.

## Tags

- `latest` — the current stable build.
- Multi-arch: published for **linux/amd64** and **linux/arm64**, so the right
  variant is pulled automatically on Intel/AMD and Apple Silicon / ARM hosts.
